# tools/ — Knowledge Index Infrastructure

<!-- DEPLOY: project-root/tools/CLAUDE.md -->

This directory contains the knowledge index pipeline — the scripts, data, and outputs that transform session results into a structured, queryable knowledge graph.

## Structure

```
tools/
├── knowledge-schema.yaml       # Entity-type definitions (drives the indexer)
├── knowledge-index.json        # Canonical knowledge graph (GENERATED by /weave --update)
├── knowledge.db                # SQLite cache (optional, REBUILT from JSON)
└── viz/                        # Generated visualizations (optional)
    ├── knowledge_graph.png
    ├── data_provenance.png
    └── knowledge_graph.mmd     # Mermaid source
```

Optional Python accelerators (`extract_entities.py`, `knowledge_db.py`, `visualize_knowledge.py`) may appear here for projects that benefit from a persistent SQLite index or large-scale visualization. The core knowledge system works without them — the indexer agent reads the schema and session files directly.

## The Knowledge Pipeline

```
sessions/*.md              ─┐

{{endif-compute}}

                                    │                       │

                                    └── /weave --db-sync ─▶ knowledge.db
```

## The Single Source of Truth

**`knowledge-index.json` is the canonical knowledge graph.** Everything else is derived from it.

- SQLite (`knowledge.db`) is a **query accelerator** — rebuilt via `/weave --db-sync`
- Visualizations in `viz/` are **generated outputs** — rebuilt via `/weave --viz-all`
- Neither the database nor the visualizations are authoritative

## Entity Types

The entity types this project tracks are defined in `tools/knowledge-schema.yaml`. Inspect that file — or run `/weave --show` — to see the current set. A universal baseline of 5 types (`sessions`, `researchers`, `results`, `references`, `open_questions`) is always present; discipline packs may add domain-specific types (e.g., `theorems` and `conjectures` for math, `gates` and `constants` for physics).

## Who Writes What

| Actor | Can Do |
|:------|:-------|
| `indexer` agent | **Sole writer** of `knowledge-index.json` |
| `/weave --update` command | Triggers the indexer to rebuild |
| Everyone else | **Read only** — query via `/weave` |

## Source Authority Hierarchy

When sources conflict, the ordering defined in `tools/knowledge-schema.yaml` (`authority_order`) is canonical. The universal default runs: skeptic verdicts → coordinator records → librarian metadata → indexer extractions → raw session/data files (lowest). Discipline packs may override this ordering.

## Rules

- **Never hand-edit `knowledge-index.json`** — always use `/weave --update`
- **Never treat `knowledge.db` as source of truth** — it's a cache
- **Run `/weave --update` after editing session files** — the hook in `settings.local.json` reminds you
- **Entity extraction is schema-driven** — the indexer reads `knowledge-schema.yaml` and extracts per spec; no hardcoded regex
- **Visualizations are disposable** — regenerate with `/weave --viz-all`
