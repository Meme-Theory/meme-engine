# sessions/framework/ — Cross-Session Framework Documents

<!-- DEPLOY: project-root/sessions/framework/CLAUDE.md -->

This directory contains documents that span multiple sessions — mechanism discussions, theoretical frameworks, evolving models, and synthesis documents that accumulate across the project lifecycle.

## What Belongs Here

- **Mechanism discussions** — documents exploring specific theoretical mechanisms across multiple sessions
- **Framework evolution** — how the project's central ideas develop over time
- **Cross-session synthesis** — documents that consolidate results from multiple sessions
- **Methodology documents** — project-wide analytical frameworks

## What Does NOT Belong Here

- **Single-session outputs** — those go in `sessions/session-NN/`
- **Planning documents** — those go in `sessions/session-plan/`
- **Reference papers** — those go in `researchers/`
- **The constraint map** — that lives in `.claude/agent-memory/constraint-map.md`

## Rules

- **Latest synthesis wins** — when multiple documents address the same topic, the most recent is canonical
- **Link to source sessions** — every claim must reference the session where it was established
- **Framework documents evolve** — unlike session outputs (which are snapshots), these are living documents
- **Coordinator maintains** — the coordinator agent is responsible for keeping these documents current
